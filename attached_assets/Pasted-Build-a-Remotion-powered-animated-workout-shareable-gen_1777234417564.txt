Build a Remotion-powered animated workout shareable generator for my private fitness app.

Context:
I already have workout activities with:
- title
- sport type
- date
- duration
- distance
- pace
- elevation
- GPS route points as lat/lng/time
- optional image or map background

Goal:
Create a system that generates a polished 10–15 second MP4 shareable asset for Instagram Stories/Reels from an activity.

Important architecture:
Cloudflare Workers should NOT do the actual video rendering. Use Cloudflare Workers only as the lightweight API/job trigger layer.
The Remotion render should happen in a Node.js backend/render worker because Remotion rendering needs Chromium/FFmpeg-style server execution.

Build this architecture:

1. Frontend
- Add a “Generate Video” button on the activity page.
- When clicked, POST to `/api/render-video` with the activity ID.
- Show render status: queued, rendering, complete, failed.
- When complete, show the MP4 preview and download/share link.

2. Cloudflare Worker API
Create a Cloudflare Worker that exposes:
- `POST /render-video`
- `GET /render-status/:jobId`

The worker should:
- validate the request
- create a render job
- store job metadata
- forward the job to the render backend
- return `{ jobId, status: "queued" }`

Use Cloudflare KV or D1 for lightweight job status storage.

3. Render backend
Create a Node.js render service that uses Remotion.

It should:
- receive a render job
- load the activity data
- render a Remotion composition to MP4
- save the finished video to storage
- update job status to complete with the video URL

4. Remotion animation
Create a Remotion composition called `WorkoutRouteShareable`.

Format:
- 1080x1920 vertical video
- 30fps
- 12 seconds duration
- dark premium design
- suitable for Instagram Stories/Reels

Animation sequence:
0–1.5s:
- Fade in app name/logo
- Show workout title and date

1.5–7s:
- Animate the GPS route drawing itself over a dark map-style background
- A small glowing dot should travel along the route
- Route should be normalized to fit nicely inside the video frame
- Add subtle motion/zoom so it feels alive

7–10s:
- Stats animate in:
  - distance
  - duration
  - average pace
  - elevation gain
- Values should count up smoothly

10–12s:
- Final polished share card:
  - full route visible
  - title
  - key stats
  - small “Generated by [app name]” footer

5. Route animation requirements
- Convert lat/lng points into normalized x/y coordinates.
- Preserve aspect ratio.
- Add padding around the route.
- Draw the route progressively based on animation progress.
- Use SVG polyline or path.
- Animate the route using Remotion interpolate/useCurrentFrame.
- Add a moving dot at the current route position.

6. Code quality
- Use TypeScript.
- Keep components clean and reusable.
- Add comments where the rendering pipeline is non-obvious.
- Include clear setup instructions.
- Include environment variables needed:
  - DATABASE_URL
  - STORAGE_BUCKET
  - RENDER_BACKEND_URL
  - CLOUDFLARE_KV_NAMESPACE or D1 binding
  - PUBLIC_BASE_URL

7. Storage
For now, save rendered MP4s locally in development.
Structure it so production storage can later be swapped to Cloudflare R2.

8. Deliverables
Create:
- Remotion composition
- route normalization utility
- animated stat component
- render backend endpoint
- Cloudflare Worker endpoint
- frontend Generate Video button
- job status polling
- README with exact commands to run locally

Make the first version simple and working before adding complexity.
Prioritize a beautiful route animation and clean vertical video output.
